## Task 3.1: Beta Manual — 30 Early Adopters

**Mengapa:** Ini validasi PALING PENTING sebelum launch. 30 orang tua yang benar-benar pakai produk = data real, bukan tebakan.

### Langkah detail:

#### Rekrut (Hari 1-2):
- [ ] Buka waiting list email → kirim email: "Kami mencari 30 early adopter untuk mencoba Kepompong"
- [ ] Posting di grup WA/FB: "Beta tester Kepompong — gratis 1 bulan, feedback dihargai"
- [ ] Target 30 orang. Kalau kurang dari waiting list, ajak kenalan/teman.
- [ ] Buat grup WA "Beta Kepompong — Early Adopters"

#### Kirim aktivitas (Hari 1-7):
- [ ] Setiap pagi (jam 06.00-07.00), kirim 1 aktivitas ke grup WA:
  ```
  ☀️ Selamat pagi, Bunda!
  Aktivitas hari ini:
  [Nama Aktivitas]
  Usia: [usia]
  Durasi: [menit]
  Bahan: [bahan]
  Cara: [langkah singkat]
  Soft skill: [karakter yang dilatih]
  
  Coba yuk, lalu cerita hasilnya di sini! 👇
  ```
- [ ] Aktivitas bisa ambil dari database yang sudah ada

#### Kumpulkan feedback (Hari 1-7):
- [ ] Tanyakan: "Aktivitasnya dicoba? Berapa lama? Anaknya suka?"
- [ ] Catat: aktivitas favorit, durasi main, kesulitan
- [ ] Catat di spreadsheet:
  ```
  | Nama User | Aktivitas | Dicoba? | Durasi | Feedback |
  |-----------|-----------|---------|--------|----------|
  ```

#### Sean Ellis Test (Hari 5-7):
- [ ] Kirim poll ke grup WA: "How would you feel if Kepompong disappeared?"
  - Sangat kecewa (Very Disappointed)
  - Agak kecewa (Somewhat Disappointed)
  - Tidak kecewa (Not Disappointed)
  - Tidak pakai (Don't use)
- [ ] Target: **Very Disappointed ≥30%** (ini indikator PMF)
- [ ] Kalau <30%, tanya follow-up: "Apa yang kurang?"

## Task 3.2: Final Testing MVP

**Mengapa:** Launch day bukan hari untuk pertama kali nyoba fitur. Semua harus sudah di-test.

### Langkah detail:

- [ ] Test flow lengkap:
  ```
  1. Buka landing page → klik "Daftar"
  2. Register (email + password)
  3. Onboarding: input nama anak, usia, minat
  4. Dapat rekomendasi aktivitas (3 detik?)
  5. Klik aktivitas → lihat detail
  6. Tandai "Selesai"
  7. Lihat progress dashboard
  8. Coba assessment
  9. Coba challenge system
  ```
- [ ] Test di 3 device berbeda:
  - HP Android (Chrome)
  - iPhone (Safari)
  - Desktop (Chrome)
- [ ] Catat setiap error yang muncul → fix segera
- [ ] Test kecepatan: rekomendasi muncul <3 detik? Kalau lebih, catat.
- [ ] Test offline mode: worksheet bisa didownload & dicetak?
- [ ] Fix semua bug blocker (error yang bikin user ga bisa lanjut)
- [ ] Catat bug minor di list "Post-Launch Fix" — jangan tunda launch karena bug minor

## Task 3.3: Payment System Integration

**Mengapa:** Launch = jualan. Jualan = payment harus jalan.

### Langkah detail:

- [ ] Integrasi payment gateway:
  - Pilih: **Midtrans** (lebih populer di Indonesia) atau **Xendit**
  - Daftar akun merchant (butuh dokumen usaha — siapkan lebih awal)
  - Integrasi API ke backend Laravel
- [ ] Payment methods:
  - **QRIS** (wajib — 80% transaksi digital Indonesia)
  - Virtual Account (BCA, Mandiri, BRI)
  - E-Wallet (GoPay, OVO, Dana) — via QRIS sudah mencakup
- [ ] Setup pricing plans di backend:
  - Plan 1: Bulanan (RpXX.XXX)
  - Plan 2: 3 Bulan (RpXX.XXX — diskon)
  - Plan 3: Tahunan (RpXX.XXX — diskon terbesar)
- [ ] Setup trial flow:
  - Register → trial 7 hari gratis → setelah H+7 minta bayar
- [ ] Test flow payment:
  ```
  1. Register akun baru
  2. Trial starts (Hari 1)
  3. Coba akses fitur premium (harus bisa)
  4. Buka halaman pricing → klik "Langganan"
  5. Pilih payment method → bayar (QRIS)
  6. Konfirmasi pembayaran → status "Active"
  7. Cek di dashboard: sisa hari bertambah
  8. Test expire trial → akses fitur premium hilang
  9. Test cancel subscription
  ```

## Task 3.4: Final Pricing

**Mengapa:** Data dari fake door test (Week 2) akan menentukan harga final.

### Langkah detail:

- [ ] Analisis hasil fake door test:
  ```
  Varian A (Rp49K): X clicks, Y% conversion
  Varian B (Rp79K): X clicks, Y% conversion
  Varian C (Rp129K): X clicks, Y% conversion
  
  - Varian mana paling banyak diklik? → indikasi preferred price
  - Varian mana konversi tertinggi? → indikasi sweet spot
  ```
- [ ] Tentukan harga final:
  - Contoh: Rp49K/bulan (entry), Rp129K/3 bulan (value), Rp399K/tahun (best value)
  - Atau: Rp79K/bulan (single), Rp199K/3 bulan (family up to 5)
- [ ] Tentukan annual discount (contoh: beli tahunan = hemat 33%)
- [ ] Siapkan promo launch:
  - "Early Bird Diskon 40% — 100 pendaftar pertama"
  - "Launch Week — harga khusus, naik setelah 1 minggu"
- [ ] Update pricing di backend Laravel (via admin panel atau langsung DB)
- [ ] Update pricing di landing page website:
  - Buka `D:\kepompong\website\src\sections\Pricing.astro`
  - Ganti dengan harga final
  - Pastikan CTA mengarah ke register/payment (bukan waiting list lagi)
- [ ] Test: klik pricing → register → bayar → flow beres
